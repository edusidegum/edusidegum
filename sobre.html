Bacharel em Sistemas de Informação — uma formação que não ensina uma linguagem específica, mas ensina a pensar sistemas completos: como os dados se organizam, como as páginas se conectam, como a informação flui entre o usuário e o negócio.

Isso é arquitetura, não implementação.

O Que Isso Significa na Prática

Três tempos de um mesmo projeto

Minha atuação se aproxima do que o mercado reconhece como Arquiteto de Informação ou Arquiteto de Soluções Digitais — o profissional que realiza o briefing, desenha o projeto, define as conexões e funcionalidades, e entrega uma solução ajustada à realidade da empresa, sem exigir que ela altere seus processos internos para se adaptar à tecnologia.

01 / Diagnóstico

Briefing e Diagnóstico

Antes de qualquer definição técnica, o trabalho é de escuta e organização. Onde estão os dados? Como o cliente encontra o que precisa? Qual o fluxo atual — e qual deveria ser? O entregável não é código, é clareza.

02 / Estrutura

Desenho da Solução

Com o diagnóstico feito, o projeto é estruturado: mapa do site, conexões entre páginas, funcionalidades necessárias, integrações com ferramentas que a empresa já utiliza. O objetivo é que a solução se encaixe no negócio, e não o contrário.

03 / Ponte

Conexão com a Implementação

O projeto executivo documentado serve de ponte entre a necessidade do cliente e quem vai implementar — seja uma equipe técnica, um profissional selecionado para a execução, ou o próprio cliente com ferramentas adequadas. Meu papel é garantir que o que foi desenhado seja executável, não necessariamente executá-lo pessoalmente em cada camada.
